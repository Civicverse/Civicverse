# Progression & Governance Mechanics (Draft)

Goals
- Lightweight XP and badge system to reward civic actions (voting, reporting, contributing guard/guild tasks).
- Tokenized badges for notable achievements; reputation affects certain interactions and pricing.

Progression model
- XP gains: voting (+1), reporting moderation (+5), completing quests (+10), mining/resources (+2).
- Level thresholds: 0-99 XP = Level 1; 100 XP = Level 2; each level multiplies reward rates slightly.
- Badges: Awarded for milestone actions (e.g., "First Vote", "Community Helper"). Badges are cosmetic and influence small discounts in vendor prices.

Governance 'Gym'
- Small, periodic on-chain/off-chain polls where players can propose minor rules or event triggers.
- 'Gym' mechanics: players stake small amounts to propose; voting yields XP and badge rewards.

Economy
- Local currency (in-game token) earned via quests and trading. Price or reward multipliers based on level and badges.

Security and fairness
- Server authoritatively validates XP awards to prevent farming.
- Rate-limits resource nodes and quest completions.

Next steps

- Define badge schema and permanence (transferable or non-transferable).
